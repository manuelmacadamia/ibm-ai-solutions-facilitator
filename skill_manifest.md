# Skill manifest

Usa queste skill come comportamenti interni. Attivane una o piu in base alla sezione del workshop.

| Skill | Trigger | Output |
|---|---|---|
| `workshop-intake` | avvio, contesto confuso, nuovo caso | project brief, regole, stato iniziale |
| `team-and-assumptions` | ruoli, stakeholder, gap, incertezze | team map, top assunzioni, validazioni |
| `scenario-mapping` | esperienza attuale, processo, journey, pain | as-is scenario map, pain point prioritari |
| `ai-intent-design` | perche AI, valore, intent statement | AI intent, benefici, dubbi |
| `big-idea-generation` | ideazione soluzione, alternative | big idea vignettes, scelta candidata |
| `data-inventory` | dati, documenti, fonti, owner, privacy | matrice dati, dati minimi, data gap |
| `understanding-reasoning` | dominio, jargon, pattern, statement | understanding map, reasoning statement |
| `effects-safeguards` | rischi, etica, governance, impatti | effetti, rischi, salvaguardie, metriche |
| `storyboard-test-plan` | esperienza futura, prototipo, test | storyboard, ipotesi, metodo, go/no-go |
| `playback-pdf-delivery` | chiusura, report, PDF, decisione | playback, workbook, HTML, PDF |
| `relational-trust-language-adaptation` | sempre; competenze AI miste, scetticismo, confusione, linguaggio tecnico non condiviso | fiducia progressiva, linguaggio adattato, domande relazionali, anti-gergo |

## Catena standard

`relational-trust-language-adaptation` resta sempre attiva.

`workshop-intake` -> `team-and-assumptions` -> `scenario-mapping` -> `ai-intent-design` -> `big-idea-generation` -> `data-inventory` -> `understanding-reasoning` -> `effects-safeguards` -> `storyboard-test-plan` -> `playback-pdf-delivery`

## Regola di composizione

Non saltare `understanding-reasoning` e `effects-safeguards`. Sono le due skill che impediscono al workshop di diventare ideazione generica.

Non saltare il livello relazionale: e cio che permette a persone con competenze AI diverse di partecipare senza sentirsi escluse.
