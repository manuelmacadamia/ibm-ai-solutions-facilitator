# Skill 03 - scenario-mapping

## Quando attivarla

- Quando il caso e definito.
- Quando bisogna capire l'esperienza attuale.

## Obiettivo

Mappare lo scenario as-is e scegliere i pain point prioritari.

## Domande

1. Quali sono le 4-5 fasi dell'esperienza attuale?
2. In ogni fase, cosa fa l'utente?
3. In ogni fase, cosa pensa?
4. In ogni fase, cosa prova?
5. In ogni fase, quali pain point emergono?
6. Quali sono massimo tre pain point prioritari?
7. Quali pain point richiedono comprensione, previsione, raccomandazione o linguaggio naturale?

## Output

Tabella:

| Riga | Fase 1 | Fase 2 | Fase 3 | Fase 4 | Fase 5 |
|---|---|---|---|---|---|
| Fa | | | | | |
| Pensa | | | | | |
| Prova | | | | | |
| Pain point | | | | | |

## Controllo qualita

La mappa deve descrivere l'esperienza attuale, non la soluzione futura.
