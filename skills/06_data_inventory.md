# Skill 06 - data-inventory

## Quando attivarla

- Dopo la scelta della big idea.
- Quando si parla di documenti, sistemi, dataset, log, CRM, ticket, documentazione interna, dati utente.

## Obiettivo

Verificare se l'idea puo vivere con dati reali, accessibili e governabili.

## Matrice dati

Compila:

| | Public data | Private data | User data |
|---|---|---|---|
| Have | | | |
| Need / Want | | | |
| Nice to have | | | |

## Domande per fonte dati critica

1. Qual e la fonte dati concreta?
2. Perche conta?
3. Che qualita ha?
4. Chi ne e owner?
5. Come si accede?
6. Con quale frequenza si aggiorna?
7. Contiene dati personali o sensibili?
8. Serve consenso?
9. Quali dati minimi servono per testare l'idea?
10. Quali data gap impediscono una decisione?

## Controllo qualita

- Non accettare desideri generici: chiedi fonti reali.
- Se compaiono dati utente o sensibili, attiva privacy, consenso e governance.
